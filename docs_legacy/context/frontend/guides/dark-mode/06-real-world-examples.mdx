## Example 1: Banco de Ideias - Category Cards

**Feature**: [Idea Bank](../../features/feature-idea-bank.md)  
**Component**: Category badge/card system

### Configuration Object

<details>
<summary>Show full configuration</summary>

```tsx
const CATEGORY_CONFIG = {
    documentacao: {
        label: "Documentação",
        icon: FileText,
        bgColor:
            "bg-blue-50 dark:bg-blue-950 border-blue-200 dark:border-blue-800",
        color: "text-blue-600 dark:text-blue-400",
    },
    "coleta-dados": {
        label: "Coleta de Dados",
        icon: Database,
        bgColor:
            "bg-green-50 dark:bg-green-950 border-green-200 dark:border-green-800",
        color: "text-green-600 dark:text-green-400",
    },
    "analise-ml": {
        label: "Análise ML",
        icon: Brain,
        bgColor:
            "bg-purple-50 dark:bg-purple-950 border-purple-200 dark:border-purple-800",
        color: "text-purple-600 dark:text-purple-400",
    },
    "gestao-riscos": {
        label: "Gestão de Riscos",
        icon: Shield,
        bgColor: "bg-red-50 dark:bg-red-950 border-red-200 dark:border-red-800",
        color: "text-red-600 dark:text-red-400",
    },
};
```

</details>

### Usage in Component

```tsx
function IdeaCard({ idea }) {
    const config = CATEGORY_CONFIG[idea.category];

    return (
        <div className="rounded-lg border bg-white dark:bg-gray-900 p-4">
            <div
                className={`inline-flex items-center gap-2 rounded-md border px-3 py-1 ${config.bgColor}`}
            >
                <config.icon className={`h-4 w-4 ${config.color}`} />
                <span className={`text-sm font-medium ${config.color}`}>
                    {config.label}
                </span>
            </div>

            <h3 className="mt-3 text-lg font-semibold text-gray-900 dark:text-white">
                {idea.title}
            </h3>

            <p className="mt-2 text-sm text-gray-600 dark:text-gray-400">
                {idea.description}
            </p>
        </div>
    );
}
```

**Key Patterns:**

-   Configuration object centralizes all color variants
-   Light mode uses 50/200/600 shades
-   Dark mode uses 950/800/400 shades
-   Icons and text use same color scale

## Example 2: EscopoPage - Tech Stack Cards

**Feature**: Project scope page  
**Component**: Technology stack cards with colored backgrounds

### Color Configuration

<details>
<summary>Show full configuration</summary>

```tsx
const colorClasses = {
    green: "border-green-200 bg-green-50 dark:border-green-900 dark:bg-green-950",
    purple: "border-purple-200 bg-purple-50 dark:border-purple-900 dark:bg-purple-950",
    orange: "border-orange-200 bg-orange-50 dark:border-orange-900 dark:bg-orange-950",
    blue: "border-blue-200 bg-blue-50 dark:border-blue-900 dark:bg-blue-950",
};

const textClasses = {
    green: "text-green-900 dark:text-green-100",
    purple: "text-purple-900 dark:text-purple-100",
    orange: "text-orange-900 dark:text-orange-100",
    blue: "text-blue-900 dark:text-blue-100",
};
```

</details>

### Usage in Component

```tsx
function TechStackCard({ title, description, color }) {
    return (
        <div className={`rounded-lg border-2 p-6 ${colorClasses[color]}`}>
            <h3 className={`text-xl font-bold ${textClasses[color]}`}>
                {title}
            </h3>
            <p className={`mt-2 ${textClasses[color]}`}>{description}</p>
        </div>
    );
}

// Usage
<TechStackCard
    title="Backend"
    description=".NET 8 + Python 3.11"
    color="green"
/>;
```

**Key Patterns:**

-   Separate configuration for backgrounds and text
-   Consistent shade mapping (50→950, 200→900, 900→100)
-   Reusable across multiple card types

## Example 3: Alert/Warning Boxes

**Feature**: System-wide alerts  
**Component**: Alert messages with semantic colors

### Critical Alerts

<details>
<summary>Show critical alert component</summary>

```tsx
function CriticalAlert({ message }) {
    return (
        <div
            className="rounded-lg border-2 border-red-300 bg-red-50 p-4
                    dark:border-red-800 dark:bg-red-950"
        >
            <div className="flex items-start gap-3">
                <AlertTriangle className="h-6 w-6 text-red-600 dark:text-red-400" />
                <div>
                    <h4 className="font-semibold text-red-800 dark:text-red-300">
                        Critical Warning
                    </h4>
                    <p className="mt-1 text-sm text-red-700 dark:text-red-400">
                        {message}
                    </p>
                </div>
            </div>
        </div>
    );
}
```

</details>

### Success Messages

```tsx
function SuccessMessage({ message }) {
    return (
        <div
            className="rounded-lg border border-green-200 bg-green-50 p-4
                    dark:border-green-800 dark:bg-green-950"
        >
            <div className="flex items-center gap-2">
                <CheckCircle className="h-5 w-5 text-green-600 dark:text-green-400" />
                <p className="text-sm text-green-800 dark:text-green-300">
                    {message}
                </p>
            </div>
        </div>
    );
}
```

### Info Messages

```tsx
function InfoMessage({ message }) {
    return (
        <div
            className="rounded-lg border border-blue-200 bg-blue-50 p-4
                    dark:border-blue-800 dark:bg-blue-950"
        >
            <div className="flex items-center gap-2">
                <Info className="h-5 w-5 text-blue-600 dark:text-blue-400" />
                <p className="text-sm text-blue-800 dark:text-blue-300">
                    {message}
                </p>
            </div>
        </div>
    );
}
```

**Key Patterns:**

-   Semantic colors (red=error, green=success, blue=info)
-   Consistent shade mapping across all alert types
-   Icons use same color scale as text
-   Border thickness varies by severity (2px for critical, 1px for others)

## Example 4: Form Elements

**Feature**: System-wide forms  
**Component**: Input, select, textarea with dark mode

### Text Input

<details>
<summary>Show text input component</summary>

```tsx
function TextInput({ label, value, onChange, placeholder }) {
    return (
        <div>
            <label className="block text-sm font-medium text-gray-700 dark:text-gray-300">
                {label}
            </label>
            <input
                type="text"
                value={value}
                onChange={onChange}
                placeholder={placeholder}
                className="mt-1 w-full rounded-md border border-gray-300 dark:border-gray-600
                   bg-white dark:bg-gray-800
                   px-3 py-2 text-sm
                   text-gray-900 dark:text-gray-100
                   placeholder:text-gray-400 dark:placeholder:text-gray-500
                   focus:border-cyan-500 dark:focus:border-cyan-400
                   focus:outline-none focus:ring-1 focus:ring-cyan-500 dark:focus:ring-cyan-400"
            />
        </div>
    );
}
```

</details>

### Select Dropdown

<details>
<summary>Show select dropdown component</summary>

```tsx
function SelectInput({ label, value, onChange, options }) {
    return (
        <div>
            <label className="block text-sm font-medium text-gray-700 dark:text-gray-300">
                {label}
            </label>
            <select
                value={value}
                onChange={onChange}
                className="mt-1 w-full rounded-md border border-gray-300 dark:border-gray-600
                   bg-white dark:bg-gray-800
                   px-3 py-2 text-sm
                   text-gray-900 dark:text-gray-100
                   focus:border-cyan-500 dark:focus:border-cyan-400
                   focus:outline-none focus:ring-1 focus:ring-cyan-500 dark:focus:ring-cyan-400"
            >
                {options.map((opt) => (
                    <option key={opt.value} value={opt.value}>
                        {opt.label}
                    </option>
                ))}
            </select>
        </div>
    );
}
```

</details>

**Key Patterns:**

-   Labels use gray-700/gray-300 for readability
-   Inputs use white/gray-800 backgrounds
-   Borders use gray-300/gray-600
-   Focus states use cyan accent color
-   Placeholder text uses muted gray

## Example 5: Data Tables

**Feature**: [TP Capital Signals](../../features/feature-tp-capital-signals.md)  
**Component**: QuestDB data table with alternating rows

### Table Structure

<details>
<summary>Show full table component</summary>

```tsx
function DataTable({ data }) {
    return (
        <div className="overflow-x-auto rounded-lg border border-gray-200 dark:border-gray-700">
            <table className="w-full">
                <thead className="bg-gray-50 dark:bg-gray-900">
                    <tr>
                        <th className="px-4 py-3 text-left text-sm font-semibold text-gray-900 dark:text-gray-100">
                            Symbol
                        </th>
                        <th className="px-4 py-3 text-left text-sm font-semibold text-gray-900 dark:text-gray-100">
                            Signal
                        </th>
                        <th className="px-4 py-3 text-left text-sm font-semibold text-gray-900 dark:text-gray-100">
                            Price
                        </th>
                    </tr>
                </thead>
                <tbody className="divide-y divide-gray-200 dark:divide-gray-700">
                    {data.map((row, idx) => (
                        <tr
                            key={row.id}
                            className={
                                idx % 2 === 0
                                    ? "bg-white dark:bg-gray-950"
                                    : "bg-gray-50 dark:bg-gray-900"
                            }
                        >
                            <td className="px-4 py-3 text-sm text-gray-900 dark:text-gray-100">
                                {row.symbol}
                            </td>
                            <td className="px-4 py-3 text-sm text-gray-900 dark:text-gray-100">
                                {row.signal}
                            </td>
                            <td className="px-4 py-3 text-sm text-gray-900 dark:text-gray-100">
                                {row.price}
                            </td>
                        </tr>
                    ))}
                </tbody>
            </table>
        </div>
    );
}
```

</details>

**Key Patterns:**

-   Table header uses gray-50/gray-900 background
-   Alternating row colors for readability
-   Dividers use gray-200/gray-700
-   Border around entire table

## Example 6: Status Badges

**Feature**: System-wide status indicators  
**Component**: Status badges with semantic colors

### Status Configuration

<details>
<summary>Show status configuration</summary>

```tsx
const STATUS_CONFIG = {
    active: {
        label: "Active",
        bgColor: "bg-green-50 dark:bg-green-950",
        textColor: "text-green-700 dark:text-green-300",
        dotColor: "bg-green-500 dark:bg-green-400",
    },
    pending: {
        label: "Pending",
        bgColor: "bg-yellow-50 dark:bg-yellow-950",
        textColor: "text-yellow-700 dark:text-yellow-300",
        dotColor: "bg-yellow-500 dark:bg-yellow-400",
    },
    error: {
        label: "Error",
        bgColor: "bg-red-50 dark:bg-red-950",
        textColor: "text-red-700 dark:text-red-300",
        dotColor: "bg-red-500 dark:bg-red-400",
    },
};
```

</details>

### Usage in Component

```tsx
function StatusBadge({ status }) {
    const config = STATUS_CONFIG[status];

    return (
        <span
            className={`inline-flex items-center gap-1.5 rounded-full px-2.5 py-0.5 text-xs font-medium ${config.bgColor} ${config.textColor}`}
        >
            <span className={`h-1.5 w-1.5 rounded-full ${config.dotColor}`} />
            {config.label}
        </span>
    );
}
```

**Key Patterns:**

-   Pill-shaped badges with rounded-full
-   Colored dot indicator
-   Semantic colors for different statuses
-   Consistent shade mapping
