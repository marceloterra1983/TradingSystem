Introduction to TradingSystem

Last updated: October 19, 2025

Welcome to TradingSystem documentation. This guide will help you understand and work with our trading platform built with Clean Architecture and DDD principles.

Note

This documentation is for TradingSystem v2.0+. For older versions, see the legacy docs.

What is TradingSystem?

TradingSystem is a comprehensive trading platform that integrates with Nelogica's ProfitDLL for market data capture, ML-based analysis, and automated order execution. The system is designed to run 100% on-premise with no cloud dependencies.

Key Features

  • Real-time Market Data - Direct integration with ProfitDLL
  • ML Analysis - Cause-and-effect model for signal generation
  • Risk Management - Built-in safety mechanisms and kill switches
  • Clean Architecture - Domain-driven design with clear separation of concerns
  • Microservices - Independent, scalable services

Getting Started

Follow these steps to set up your development environment:

Prerequisites

bash 
# Node.js 20+
node --version

# .NET 8.0
dotnet --version

# Python 3.11+
python --version

# Docker & Docker Compose
docker --version
docker compose version

Installation

bash 
# Clone the repository
git clone https://github.com/tradingsystem/tradingsystem.git
cd tradingsystem

# Install frontend dependencies
cd frontend/apps/dashboard
npm install

# Install backend dependencies
cd ../../backend/api/workspace
npm install

# Start services
bash scripts/services/start-all.sh

Warning

Core trading services MUST run natively on Windows due to ProfitDLL requirements. See the deployment guide for details.
