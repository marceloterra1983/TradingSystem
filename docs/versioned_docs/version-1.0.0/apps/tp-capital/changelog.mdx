## Releases

### v1.0.0 - 2025-10-23

- Initial production release.
- Telegram bot listener (polling mode).
- Message parser for TP Capital signal format.
- QuestDB integration via ILP protocol.
- REST API endpoints: `/signals`, `/logs`, `/telegram/*`.
- Prometheus metrics endpoint.
- Docker Compose deployment.
- GitHub Actions CI/CD pipeline.

**Known Issues**: WebSocket support pending, rate limiting not enforced, QuestDB backups manual.

### v0.9.0 - 2025-10-15 (Beta)

- Beta release for internal testing.
- QuestDB integration validated.
- Dashboard integration tested.
- Added `/signals` pagination, Telegram bot polling, Docker Compose stack, sample data seeding.
- Bug fixes: QuestDB retry logic, timestamp parsing, CORS configuration.

### v0.5.0 - 2025-10-01 (Alpha)

- Initial alpha release.
- Basic Telegram ingestion.
- QuestDB proof of concept.
- Simple REST API.

## Upcoming Releases

### v1.1.0 - Planned Q1 2025

- WebSocket support for real-time signal push.
- Telegram webhook mode.
- Rate limiting per client.
- Automated QuestDB backup.
- Enhanced error handling and retry logic.

### v1.2.0 - Planned Q2 2025

- Multi-channel support (TP Premium and others).
- Configurable signal validation rules.
- Historical data export API.
- Performance optimizations (batching).

## Migration Notes

### Upgrading from v0.9.0 to v1.0.0

- Breaking changes: None.
- Steps: Pull latest image, restart services, verify health.
- Database migrations: Not required.

### Upgrading from v0.5.0 to v0.9.0

- Breaking changes: `PORT` renamed to `TP_CAPITAL_API_PORT`, `/api/signals` renamed to `/signals`.
- Steps: Update `.env`, adjust dashboard API calls, pull image, restart.
- Database migrations: Run `npm run migrate` (adds `channel` column).

## Deprecation Notices

- Polling mode scheduled for deprecation in v1.1.0 (webhooks preferred).
- Legacy `/api/*` routes removed; use root-level routes.
