## Overview

The `order.canceled` event notifies consumers that an order was successfully canceled through the Order Manager API or a kill-switch automation. Detailed payload attributes will be finalized alongside the cancellation workflow implementation.

**Event Name**: `order.canceled`  
**Version**: v1 (draft)  
**Status**: Stub (schema pending)  
**Trigger**: Order cancellation via `DELETE /api/orders/{orderId}` or automated kill switch
