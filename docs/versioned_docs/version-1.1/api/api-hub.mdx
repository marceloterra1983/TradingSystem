## üîç Common Patterns

### Pagination

All list endpoints support pagination:

```bash
GET /api/messages?page=1&limit=20&offset=0
```

### Filtering

Use query parameters for filtering:

```bash
# Filter by channel
GET /api/messages?channelId=-1001649127710

# Filter by date range
GET /api/messages?startDate=2025-11-01&endDate=2025-11-04

# Combine filters
GET /api/messages?channelId=-1001649127710&startDate=2025-11-04&limit=50
```

### Sorting

Use `sortBy` and `sortOrder` parameters:

```bash
GET /api/messages?sortBy=created_at&sortOrder=desc
```

### Error Handling

All APIs return consistent error responses:

```json
{
  "success": false,
  "error": "ValidationError",
  "message": "Invalid channel ID format",
  "code": "INVALID_CHANNEL_ID",
  "timestamp": "2025-11-04T18:00:00.000Z"
}
```

## üöÄ Performance Tips

### 1. Use Pagination
Always use `limit` parameter to avoid large responses:
```bash
GET /api/messages?limit=20  # ‚úÖ Good
GET /api/messages            # ‚ùå May return 1000+ messages
```

### 2. Cache Responses
APIs include cache headers:
```
Cache-Control: public, max-age=60
ETag: "abc123"
```

Use `If-None-Match` header to leverage caching.

### 3. Batch Operations
When possible, use batch endpoints:
```bash
POST /api/messages/batch  # Process multiple messages
```

### 4. Use Specific Queries
Filter on the server side instead of client side:
```bash
# ‚úÖ Good: Filter server-side
GET /api/messages?channelId=123&status=unread

# ‚ùå Bad: Fetch all and filter client-side
GET /api/messages
```

## üêõ Troubleshooting

### API Not Responding

```bash
# 1. Check if service is running
lsof -i :<port>

# 2. Check health endpoint
curl http://localhost:<port>/health

# 3. Check logs
tail -f logs/<service>.log

# 4. Authentication Errors

```bash
# Verify API key in .env
grep "API_KEY\|API_TOKEN" .env

# Test with correct key
curl -H "X-API-Key: your_key" http://localhost:4010/api/messages
```

### Rate Limit Exceeded

```bash
# Check rate limit headers
curl -I http://localhost:4010/api/messages

# Wait for reset time (X-RateLimit-Reset header)
# Or request rate limit increase
```
