## Overview

This catalogue lists all domain schemas for TradingSystem v1. Schemas define the structure, validation rules, and relationships for core domain entities.

**Version**: v1  
**Status**: Specification (implementation pending)  
**Domain**: Trading Core

## Schema Conventions

**Naming**

- Use PascalCase for schema names (Order, RiskRule)
- Use camelCase for field names (orderId, dailyLossLimit)
- Use UPPER_SNAKE_CASE for enum values (BUY, SELL, PENDING)

**Types**

- Use TypeScript interfaces for schema definitions
- Use enums for fixed value sets
- Use ISO 8601 for timestamps
- Use decimal numbers for prices (2 decimal places)
- Use integers for quantities

**Validation**

- Document all validation rules (required, ranges, formats)
- Include business rules (e.g., price within Â±10% of last trade)
- Specify error messages for validation failures

**Relationships**

- Document foreign key relationships
- Specify cardinality (one-to-one, one-to-many, many-to-many)
- Link to related schemas, events, and flows
