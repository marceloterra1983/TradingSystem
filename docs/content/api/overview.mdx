| **Prometheus Client** | All APIs | Metrics export |

### API Patterns

- **Rate Limiting**: express-rate-limit (60s windows)
- **CORS**: cors middleware with origin whitelisting
- **Security**: Helmet.js for HTTP headers
- **Validation**: JSON Schema, Joi, or custom validators
- **Error Handling**: Centralized error middleware

## API Standards

All TradingSystem APIs follow these standards:

### Documentation

- âœ… OpenAPI 3.1 specification (where applicable)
- âœ… Interactive Redocusaurus documentation
- âœ… Comprehensive MDX documentation
- âœ… Code examples in multiple languages
- âœ… Complete error documentation

### Rate Limiting

- **Default:** 120-300 requests per 60-second window (varies by API)
- **Headers:** `RateLimit-Limit`, `RateLimit-Remaining`, `RateLimit-Reset`
- **Response:** `429 Too Many Requests` when exceeded
- **Bypass:** Health checks and metrics endpoints excluded

### Authentication

- **Current:** Optional API key (development mode)
- **Production:** Token-based auth (X-API-Token, X-Gateway-Token, Authorization headers)
- **Planned:** JWT tokens, OAuth2, mTLS for trading APIs

### Observability

- âœ… Prometheus metrics at `/metrics`
- âœ… Health checks at `/health`
- âœ… Structured logging (JSON via Pino)
- âœ… Request correlation IDs (where applicable)

### Versioning

- **Strategy:** URL-based versioning (`/api/v1/...`) for stable APIs
- **Deprecation:** 6-month notice period (planned)
- **Support:** N and N-1 versions maintained

## Integration Examples

### JavaScript/TypeScript

```typescript
import axios from 'axios';

// Create API clients
const statusApi = axios.create({ baseURL: 'http://localhost:3500' });
const docsApi = axios.create({ baseURL: 'http://localhost:3400' });
const workspaceApi = axios.create({ baseURL: 'http://localhost:3200' });
const tpCapitalApi = axios.create({ baseURL: 'http://localhost:4005' });
const telegramApi = axios.create({ baseURL: 'http://localhost:4010' });
const firecrawlApi = axios.create({ baseURL: 'http://localhost:3600' });

// Check system health
const status = await statusApi.get('/api/status');
console.log(`System: ${status.data.overallStatus}`);

// Get workspace items
const items = await workspaceApi.get('/api/items');
console.log(`Items: ${items.data.items.length}`);

// Get trading signals
const signals = await tpCapitalApi.get('/signals?limit=10');
console.log(`Signals: ${signals.data.data.length}`);

// Get Telegram messages
const messages = await telegramApi.get('/api/messages?synced=false');
console.log(`Unsynced messages: ${messages.data.messages.length}`);

// Scrape a webpage
const scrape = await firecrawlApi.post('/api/scrape', {
  url: 'https://example.com'
});
console.log(`Content: ${scrape.data.data.markdown}`);
```

### Python

```python
import requests

# API base URLs
STATUS_API = 'http://localhost:3500'
DOCS_API = 'http://localhost:3400'
WORKSPACE_API = 'http://localhost:3200'
TP_CAPITAL_API = 'http://localhost:4005'
TELEGRAM_API = 'http://localhost:4010'

# Check system health
status = requests.get(f'{STATUS_API}/api/status').json()
print(f"System status: {status['overallStatus']}")

# Get workspace items
items = requests.get(f'{WORKSPACE_API}/api/items').json()
print(f"Total items: {len(items['items'])}")

# Get trading signals
signals = requests.get(f'{TP_CAPITAL_API}/signals?limit=10').json()
print(f"Signals: {len(signals['data'])}")

# Get unsynced Telegram messages
messages = requests.get(f'{TELEGRAM_API}/api/messages?synced=false').json()
print(f"Unsynced messages: {len(messages['messages'])}")
```

## API Governance

### Adding New APIs

1. **Create OpenAPI Spec** (if applicable):

   ```bash
   touch docs/static/specs/new-api.openapi.yaml
   ```

2. **Configure Redocusaurus**:
   Edit `docs/docusaurus.config.js`:

   ```javascript
   specs: [
     // ... existing specs
     {
       id: 'new-api',
       spec: 'static/specs/new-api.openapi.yaml',
       route: '/api/new-api',
     },
   ]
   ```

3. **Create Documentation**:

   ```bash
   touch docs/content/api/new-api.mdx
   ```

4. **Update This Catalog**:
   Add entry to appropriate category in `overview.mdx`

5. **Add to Services Manifest**:
   Edit `config/services-manifest.json`

See `docs/static/specs/README.md` for detailed instructions.
