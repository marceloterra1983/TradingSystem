# Applications Overview

The TradingSystem consists of multiple applications organized by function and criticality.

## üìä Application Catalog

### Core Trading Infrastructure

Critical applications for market data capture and order execution.

#### Data Capture

**Purpose:** Real-time market data capture via ProfitDLL integration

**Status:** üü° Planned

**Technology:** C# (.NET 8.0), ProfitDLL (64-bit), WebSocket

**Key Features:**
- Real-time ticker and order book subscriptions
- Parquet file storage
- WebSocket streaming to consumers
- Sub-millisecond latency requirements

**Quick Links:**
- [Overview](./data-capture/overview) - Architecture and design
- [API Documentation](./data-capture/api) - WebSocket protocol
- [Configuration](./data-capture/config) - Setup and environment
- [Runbook](./data-capture/runbook) - Operations guide

Supporting infrastructure for communication, data ingestion, and orchestration.

#### Service Launcher

**Purpose:** Service orchestration and comprehensive health monitoring

**Status:** üü¢ Production

**Technology:** Node.js, Express, Bash scripts

**Key Features:**
- Multi-service health monitoring (11+ services)
- Comprehensive health checks (services + containers + databases)
- Terminal service launcher (Windows/Linux)
- Smart caching (60s TTL)
- Real-time metrics aggregation
- Rate limiting and CORS protection

#### Telegram Gateway

**Purpose:** Telegram message ingestion via MTProto protocol

**Status:** üü¢ Production

**Technology:** Node.js, GramJS (MTProto), TimescaleDB, Express

**Key Features:**
- Native MTProto connection (no Bot API)
- Channel subscription management
- Message storage and retrieval
- REST API for consumers
- Automatic message synchronization
