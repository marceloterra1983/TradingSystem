## Environment Variables

Workspace uses Vite environment variables declared in `.env` files under `frontend/apps/workspace`.

### Required Variables

| Variable | Description | Example | Default |
|----------|-------------|---------|---------|
| `VITE_WORKSPACE_API_URL` | Absolute URL for Workspace API when proxy is unavailable | `http://localhost:3200` | `/api/workspace` (Vite proxy) |

### Optional Variables

| Variable | Description | Example | Default |
|----------|-------------|---------|---------|
| `VITE_APP_ENV` | Display-only environment label for footer badges | `development` | `development` |
| `VITE_ENABLE_ANALYTICS` | Enables future telemetry hooks | `false` | `false` |

> Store `.env` files locally only; they are ignored by Git. Vite exposes variables through `import.meta.env`.

## Ports and Networking

| Component | Port | Notes |
|-----------|------|-------|
| Workspace dev server | 3900 | `npm run dev` (Vite) |
| Workspace production build | N/A | Served as static assets |
| Workspace API (backend) | 3200 | Express service at `/api/items` |
| Dashboard (iframe host) | 3103 | Embeds Workspace via `/workspace` route |

### Proxy Routing

```ts
// frontend/apps/workspace/vite.config.ts
proxy: {
  '/api/workspace': {
    target: 'http://localhost:3200',
    changeOrigin: true,
    rewrite: path => path.replace(/^\/api\/workspace/, '/api')
  }
}
```

- Standalone mode (`http://localhost:3900`) consumes the proxy path.
- Dashboard iframe inherits same proxy when running at `http://localhost:3103`.
- Production deployments should set `VITE_WORKSPACE_API_URL` to the API gateway URL if a proxy is not available.

## Runtime Configuration

- Category, priority, and status taxonomies are defined in `src/components/workspace/constants`.
- Tailwind tokens imported from `frontend/tokens` keep badge colors consistent across apps.
- Dark mode respects browser/system preference and dashboard theme sync when embedded.

## Build + Quality Gates

```bash
npm run dev          # Vite dev server (3900)
npm run build        # Production build
npm run preview      # Serve built assets locally
npm run lint         # ESLint validation
npm run lint:fix     # ESLint auto-fix
npm run type-check   # TypeScript project references
npm run test         # Vitest suite
```

- `tsconfig.json` enforces strict mode with path aliases for `src`.
- ESLint/Prettier configs resolve from repo root and shared frontend profiles.
- CI should run `lint`, `type-check`, `test`, and `build` before publishing new artifacts.

## Secrets and Storage

- No sensitive credentials are bundled with the frontend.
- Backend (`backend/api/workspace`) manages database credentials separately.
- For production, use secure secret storage (Vault, parameter store) to inject API URLs during build/publish.
