## Base URL

- Direct backend (development): `http://localhost:3200/api`
- Standalone proxy (Vite dev server): `http://localhost:3900/api/workspace`
- Dashboard proxy: `http://localhost:3103/api/workspace`

## REST Endpoints

### GET /api/items

- Returns paginated list of workspace items ordered by creation date.
- Query parameters:
  - `status` (optional) — filter by workflow status.
  - `category` (optional) — filter by domain.
  - `priority` (optional) — filter by priority level.
  - `limit`/`offset` — pagination controls.
- Response payload:

  ```json
  {
    "data": [
      {
        "id": "uuid",
        "title": "Conectar Workspace ao Telegram Gateway",
        "description": "Adicionar integração na aba Status.",
        "category": "dashboard",
        "priority": "high",
        "status": "in-progress",
        "tags": ["integration", "gateway"],
        "createdAt": "2025-10-23T12:34:00.000Z",
        "updatedAt": "2025-10-24T09:15:00.000Z"
      }
    ],
    "total": 128,
    "limit": 50,
    "offset": 0
  }
  ```

### POST /api/items

- Creates new workspace item.
- Required body fields: `title`, `description`, `category`, `priority`.
- Optional fields: `tags` (array), `status`.
- Returns created item with generated `id` and timestamps.

### PUT /api/items/:id

- Updates idea attributes.
- Supports partial updates (title, description, category, priority, status, tags).
- Used by dialogs and drag-and-drop operations.

### DELETE /api/items/:id

- Removes idea after confirmation.
- Returns 204 No Content on success.

### GET /health

- Service health check used by operations.
- Example response: `{"status":"ok","workspaceApi":"connected"}`.

## Error Handling

- Validation errors return `400` with detailed messages.
- Missing resources return `404`.
- Server issues return `500` with error identifier in body.

## Authentication

- Internal service; no authentication in v1.0.0.
- Future enhancement: API keys for external integrations.

## Rate Limiting

- Not enforced currently.
- Planned limits: 100 requests/minute per client with burst of 20 (post v1.1.0).

## Integration Examples

```typescript
import axios from 'axios';

export interface WorkspaceItem {
  id: string;
  title: string;
  description: string;
  category: string;
  priority: 'low' | 'medium' | 'high' | 'critical';
  status: 'new' | 'review' | 'in-progress' | 'completed' | 'rejected';
  tags: string[];
  createdAt: string;
  updatedAt: string;
}

export async function fetchItems(params: Record<string, string>) {
  const query = new URLSearchParams(params).toString();
  const { data } = await axios.get<{ data: WorkspaceItem[] }>(
    `/api/workspace/items${query ? `?${query}` : ''}`,
  );
  return data.data;
}
```

```typescript
import { useMutation, useQuery, useQueryClient } from '@tanstack/react-query';

export function useWorkspaceItems(filters: Record<string, string>) {
  return useQuery({
    queryKey: ['workspace-items', filters],
    queryFn: () => fetchItems(filters),
    staleTime: 15000,
  });
}

export function useUpdateItem() {
  const queryClient = useQueryClient();
  return useMutation({
    mutationFn: (payload: Partial<WorkspaceItem> & { id: string }) =>
      axios.put(`/api/workspace/items/${payload.id}`, payload),
    onSuccess: () => {
      queryClient.invalidateQueries({ queryKey: ['workspace-items'] });
    },
  });
}
```

## Database Model

- Table: `workspace.workspace_items`.
- Columns: `id`, `title`, `description`, `category`, `priority`, `status`, `tags`, `metadata`, `created_at`, `updated_at`, `created_by`, `updated_by`.
- Maintained as a TimescaleDB hypertable (monthly chunks, automatic trigger updates `updated_at`).

## Eventing

- No event streams in v1.0.0.
- Planned: Webhook notifications or SSE for real-time updates (backlog item).
