## Setup & Environment

### Q: Where should I store my passwords and API keys?

**A**: Store in `.env.local` at the project root (gitignored). Never commit secrets to Git.

**Example**:

```bash
# .env.local (gitignored)
OPENAI_API_KEY=sk-proj-real-key-here
TIMESCALEDB_PASSWORD=secure-password-here
```

**See**: [Environment Variables](./tools/security-config/env)

---

### Q: Can I create a `.env` file in my service folder?

**A**: ‚ùå NO! Use only the `.env` at the project root. Create `.env.example` as a template in your service folder, but never create local `.env` files.

**Rationale**: Centralized configuration prevents scattered secrets and simplifies management.

**See**: Configuration directory guide (migration pending)

---

### Q: How do I override a variable locally?

**A**: Add it to `.env.local` at the project root or export in your shell.

**Example**:

```bash
# .env.local
MY_SERVICE_PORT=7000
MY_SERVICE_LOG_LEVEL=debug

# Or in shell
export MY_SERVICE_PORT=7000
```

---

### Q: Where can I see all available environment variables?

**A**: Check `config/.env.defaults` - all variables are documented there with comments.

**See**: [Environment Variables Reference](./tools/security-config/env)

---

### Q: How do I add variables to CI/CD?

**A**: Configure in GitHub Secrets and reference in workflow files.

**Example**:

```yaml
# .github/workflows/deploy.yml
env:
  OPENAI_API_KEY: ${{ secrets.OPENAI_API_KEY }}
```

## Development

### Q: The dashboard doesn't start - what should I check?

**A**: Verify backend is running, check health endpoints, ensure ports are not blocked.

**Quick Checks**:

```bash
# Check backend health
curl http://localhost:3200/health

# Check if port is in use
lsof -i :3103

# Check console logs
npm --prefix frontend/dashboard run dev
```

**See**: [Dashboard Troubleshooting](./apps/workspace/operations#troubleshooting)

---

### Q: I get "EADDRINUSE: address already in use" - what next?

**A**: Find the conflicting process and kill it, or change the port.

**Solution**:

```bash
# Find process using port
lsof -i :3103

# Kill process
kill -9 <PID>

# Or change port in .env
VITE_DASHBOARD_PORT=3104
```

---

### Q: Git error "empty ident name not allowed" during commits - how to fix?

**A**: Set Git user.name and user.email.

**Solution**:

```bash
# Global configuration
git config --global user.name "Your Name"
git config --global user.email "your.email@example.com"

# Or per-repository
git config user.name "Your Name"
git config user.email "your.email@example.com"
```

**See**: Git setup guide (migration pending)

---

### Q: Build fails with OOM or memory errors - troubleshooting tips?

**A**: Increase Node.js memory limit, clear caches, check for TypeScript errors.

**Solution**:

```bash
# Increase memory
export NODE_OPTIONS="--max-old-space-size=4096"

# Clear caches
rm -rf node_modules/.cache
npm run build

# Check TypeScript errors
npm run type-check
```

**A**: Check Docker Compose logs, verify environment variables, ensure ports are available, check disk space.

```bash

# Check logs

# Check disk space

# Verify configuration
```

## Database

### Q: Database migration returns "table not found" - what's a quick check?

**A**: Verify schema names, ensure migrations executed against correct database, check user permissions.

**Solution**:

```bash
# Check current schema
psql -h localhost -p 5433 -U timescale -d tradingsystem -c "\dt"

# List migrations
psql -h localhost -p 5433 -U timescale -d tradingsystem -c "SELECT * FROM schema_migrations;"

# Verify user permissions
psql -h localhost -p 5433 -U timescale -d tradingsystem -c "\du"
```

**See**: [Database Migrations](./database/migrations)

---

### Q: How do I backup QuestDB data?

**A**: Stop service and zip data directory, or export to CSV.

**Solution**:

```bash
# Option 1: Full backup (requires downtime)
docker compose down questdb
zip -r questdb-backup-$(date +%Y%m%d).zip $QUESTDB_ROOT/db/
docker compose up -d questdb

# Option 2: CSV export (no downtime)
curl -G http://localhost:9002/exp \
  --data-urlencode "query=SELECT * FROM tp_capital_signals" \
  -o backup.csv
```

**See**: [Database Backup & Restore](./database/retention-backup)

## Security & Keys

### Q: How do I rotate API keys safely?

**A**: Generate new key, update `.env`, restart services, verify functionality, revoke old key.

**Procedure**:

1. Generate new key (OpenAI, Telegram, etc.)
2. Update `.env.local` with new key
3. Restart affected services
4. Verify services work with new key
5. Revoke old key from provider
6. Document rotation in audit log

**See**: [Security Configuration](./tools/security-config/overview)

---

### Q: File permissions error when accessing `.env` - how to fix?

**A**: Set correct permissions (600) and ensure you own the file.

**Solution**:

```bash
# Fix permissions
chmod 600 .env

# Ensure ownership
sudo chown $USER:$USER .env

# Verify
ls -la .env
# Should show: -rw------- (600)
```
