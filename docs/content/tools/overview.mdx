# Tools & Infrastructure

Comprehensive toolkit for development, deployment, monitoring, and automation of the TradingSystem platform.

## üìö Categories

### üèóÔ∏è Infrastructure & Orchestration

#### Docker Compose (`tools/compose/`)
Docker Compose orchestration files for all platform services.

**Files:**
- `docker-compose.database.yml` - Database services (TimescaleDB, QuestDB, PostgreSQL, Adminer)
- `docker-compose.apps.yml` - Application services (Workspace, TP Capital, Documentation API, Firecrawl Proxy)
- `docker-compose.apps.prod.yml` - Production application configuration
- `docker-compose.docs.yml` - Documentation services (Docusaurus)
- `docker-compose.firecrawl.yml` - Firecrawl web scraping service
- `docker-compose.infrastructure.yml` - Core infrastructure
- `docker-compose.monitoring.yml` - Monitoring stack (Prometheus, Grafana, Alertmanager, Alert Router)
- `docker-compose.timescale.yml` - TimescaleDB standalone configuration
- `docker-compose.individual.yml` - Individual service testing

**Usage:**
```bash
# Start all databases
docker compose -f tools/compose/docker-compose.database.yml up -d

# Start monitoring stack
docker compose -f tools/monitoring/docker-compose.yml up -d

# Start all application services
docker compose -f tools/compose/docker-compose.apps.yml up -d
```

#### Nginx Reverse Proxy (`tools/nginx-proxy/`)
Nginx reverse proxy configurations for routing requests to backend services.

**Features:**
- Path-based routing
- Load balancing
- SSL/TLS termination (when configured)

#### Docker Registry (`tools/registry/`)
Private Docker registry for custom images.

#### Systemd Services (`tools/systemd/`)
Linux systemd service definitions for native services (non-containerized).

### üìä Monitoring & Observability

#### Monitoring Stack (`tools/monitoring/`)

Comprehensive monitoring solution with Prometheus, Grafana, Alertmanager, and Alert Router.

**Components:**

1. **Prometheus** (Port 9090)
   - Metrics collection and storage
   - Alert rule evaluation
   - PromQL query engine
   - Config: `tools/monitoring/prometheus/prometheus.yml`
   - Rules: `tools/monitoring/prometheus/rules/`

2. **Grafana** (Port 3000)
   - Metrics visualization
   - Pre-configured dashboards
   - Dashboard provisioning
   - Dashboards: `tools/monitoring/grafana/dashboards/`
   - Config: `tools/monitoring/grafana/provisioning/`

3. **Alertmanager** (Port 9093)
   - Alert deduplication and routing
   - Notification management
   - Silencing and inhibition
   - Config: `tools/monitoring/alertmanager/`

4. **Alert Router** (Port 8080)
   - Prometheus alerts ‚Üí GitHub Issues integration
   - Automatic issue lifecycle management
   - Alert fingerprinting and deduplication
   - Source: `tools/monitoring/alert-router/src/`
   - [OpenAPI Spec](/api/alert-router)

**Start Monitoring:**
```bash
docker compose -f tools/monitoring/docker-compose.yml up -d
```

**Access:**
- Prometheus: http://localhost:9090
- Grafana: http://localhost:3000 (admin/admin)
- Alertmanager: http://localhost:9093
- Alert Router: http://localhost:8080

### ü§ñ AI & LLM Tools

#### Agno Agents (`tools/agno-agents/`)
AI agents built with Clean Architecture and Domain-Driven Design principles.

**Structure:**
- `src/application/` - Use cases and orchestration
- `src/domain/` - Business logic and entities
- `src/infrastructure/` - External integrations
- `src/interfaces/` - API endpoints and CLI
- `tests/` - Unit and integration tests

**Purpose:** Automated workflows, intelligent decision-making, task automation.

#### LangGraph (`tools/langgraph/`)
LangGraph integration for building stateful, multi-agent AI applications.

**Structure:**
- `src/application/` - LangGraph workflows
- `src/domain/` - Business entities
- `src/infrastructure/` - LLM providers, vector stores
- `src/interfaces/` - API and CLI
- `src/monitoring/` - Observability for LangGraph

**Purpose:** Complex multi-step AI workflows, agent orchestration.

#### LlamaIndex (`tools/llamaindex/`)
RAG (Retrieval-Augmented Generation) implementation using LlamaIndex.

**Services:**
- `ingestion_service/` - Document ingestion and indexing
- `query_service/` - Query processing and retrieval
- `k8s/` - Kubernetes deployments (production, staging)
- `tests/` - Service tests

**Purpose:** Document Q&A, semantic search, knowledge retrieval.

#### Ollama (`tools/ollama/`)
Local LLM runtime for running models like Llama 2, Mistral, CodeLlama without cloud dependencies.

**Purpose:** Local model inference, development without API costs, data privacy.

## üöÄ Quick Start

### Start All Infrastructure
```bash

docker compose -f tools/compose/docker-compose.database.yml up -d

# 2. Start monitoring
docker compose -f tools/monitoring/docker-compose.yml up -d
```

- **Prometheus**: http://localhost:9090
- **Adminer** (DB UI): http://localhost:8080
