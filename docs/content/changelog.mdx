## Overview

This page serves as the central changelog index for TradingSystem, tracking project-level releases and linking to component-specific changelogs.

**Format**: Based on [Keep a Changelog](https://keepachangelog.com/)  
**Versioning**: [Semantic Versioning](https://semver.org/)

## How to Update

**Guidelines**:
- Keep entries concise and focused
- Group related changes together
- Use present tense ("Add feature" not "Added feature")
- Reference issues/PRs when applicable
- Separate breaking changes with ⚠️ emoji
- Update [Unreleased] section as you make changes
- Create new version sections when releasing

**Categories**:
- **Added**: New features
- **Changed**: Changes in existing functionality
- **Deprecated**: Soon-to-be removed features
- **Removed**: Removed features
- **Fixed**: Bug fixes
- **Security**: Security updates

**Where to Document**:
- **Project-level changes**: This file (affects multiple apps or platform)
- **App-specific changes**: Per-app changelog (e.g., `apps/order-manager/changelog.mdx`)
- **Tool-specific changes**: Per-tool changelog (e.g., `tools/docusaurus/changelog.mdx`)
- **API changes**: SDD API changelog (e.g., `sdd/api/order-manager/v1/changelog.mdx`)
