## Coverage

This section documents the TradingSystem frontend architecture, components, and engineering practices.

### Key Applications

- **Dashboard** (Port 3103): Main UI built with React 18 + Vite + TypeScript
  - 6 navigation sections (Apps, Knowledge, Infrastructure, Connections, Escopo, Conex√µes)
  - Lazy-loaded pages for optimal performance
  - Zustand (global state) + TanStack Query (server state)
  - [Architecture Documentation](/frontend/architecture/dashboard)
  - [Component Guide](/frontend/engineering/components)

- **Workspace App** (Port 3900): Standalone Kanban application
  - Embeddable in Dashboard via iframe
  - Drag-and-drop task management
  - TimescaleDB backend integration

### Architecture

- **State Management**: Zustand + TanStack Query
- **Styling**: Tailwind CSS + Radix UI components
- **Performance**: Code splitting, lazy loading, virtual scrolling
- **Type Safety**: TypeScript with strict mode
- **Build Tool**: Vite 7.x with optimized chunking

## Authoring guidance

- Prefer mermaid or PlantUML diagrams stored adjacent to their source docs.
- Use callout components (coming in Phase 2.3) to highlight operational caveats.
- Cross-link to `apps` for dashboards or embedded experiences impacted by the work.
