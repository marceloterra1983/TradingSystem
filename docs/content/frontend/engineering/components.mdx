## Core Components

### CustomizablePageLayout

**Purpose:** Drag-and-drop multi-column layout system

**Location:** `frontend/dashboard/src/components/layout/CustomizablePageLayout.tsx`

**Props:**
```typescript
interface Props {
  pageId: string;           // Unique page identifier
  title: string;            // Page title
  subtitle?: string;        // Optional subtitle
  sections: Section[];      // Content sections
  defaultColumns?: 1 | 2 | 3 | 4; // Default grid columns
}
```

**Features:**
- Drag-and-drop rearrangement (@dnd-kit)
- Multi-column grid layout
- Section collapse/expand
- Persistent state (localStorage)
- Collapse/Expand all button

**Usage Example:**
```typescript
const sections = [
  { id: 'services', content: <LocalServicesSection /> },
  { id: 'containers', content: <DockerContainersSection /> }
];

<CustomizablePageLayout
  pageId="launcher"
  title="Service Launcher"
  sections={sections}
  defaultColumns={2}
/>
```

## Component Patterns

### 1. Lazy Loading Pattern

```typescript
const ComponentName = React.lazy(() => import('./ComponentName'));

// In render:
<Suspense fallback={<Spinner />}>
  <ComponentName />
</Suspense>
```

### 2. Custom Hook Pattern

```typescript
// hooks/useServiceHealth.ts
export function useServiceHealth(serviceId: string) {
  return useQuery({
    queryKey: ['service', serviceId],
    queryFn: () => fetchHealth(serviceId),
    refetchInterval: 30000,
  });
}
```

### 3. Zustand Store Pattern

```typescript
interface Store {
  items: Item[];
  addItem: (item: Item) => void;
}

export const useStore = create<Store>((set) => ({
  items: [],
  addItem: (item) => set((state) => ({
    items: [...state.items, item]
  })),
}));
```

## Best Practices

### Component Organization

```
components/
├── ui/              # Reusable UI components (Radix wrappers)
├── layout/          # Layout components
├── pages/           # Page-level components
│   └── workspace/   # Page-specific subcomponents
└── shared/          # Shared utilities
```

### Naming Conventions

- **Components**: PascalCase (`ServiceStatusBanner`)
- **Files**: PascalCase (`ServiceStatusBanner.tsx`)
- **Props**: camelCase (`onRetry`, `isOpen`)
- **Hooks**: camelCase with `use` prefix (`useServiceHealth`)

### Type Safety

Always define prop interfaces:

```typescript
interface ComponentProps {
  title: string;
  onSave: (data: Data) => void;
  optional?: boolean;
}

export function Component({ title, onSave, optional = false }: ComponentProps) {
  // Implementation
}
```

## Performance Tips

1. **Memoization**: Use `React.memo()` for expensive components
2. **Callback Optimization**: Use `useCallback()` for event handlers
3. **Virtual Scrolling**: Use `@tanstack/react-virtual` for large lists
4. **Code Splitting**: Lazy load heavy components
5. **Query Caching**: Configure appropriate `staleTime` and `cacheTime`
